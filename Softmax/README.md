# Softmax

Softmax normalizes an input vector into a probability distribution: `softmax(x)_i = exp(x_i) / sum_j exp(x_j)`. It is a fundamental building block in Transformer attention layers (normalizing QK^T scores before the weighted sum with V), classification heads, and any multi-class output layer. In the Transformer, it is the normalization step in `Attention(Q,K,V) = softmax(QK^T / sqrt(d_k)) V`, making it one of the most frequently executed operations in modern LLM inference.

This project implements three variants of the softmax kernel targeting an RTX 3060 (GA106, Ampere) with N=4096 rows, D=8192 columns, and float32 precision: a naive 3-pass baseline, an online 1-pass optimized variant, and a warp-shuffle variant for small dimensions. The naive kernel performs three separate HBM passes per row (find max, compute exp+sum, normalize), the online kernel merges all three into a single pass, and the warp kernel uses `__shfl_xor_sync` for dimensions <= 32.

The core optimization -- the online softmax algorithm -- achieves a 40% reduction in kernel time (2.17 ms to 1.31 ms) by reducing HBM traffic from 3x reads + 1x write to 1x read + 1x write. This moves bandwidth utilization from 54.3% to 89.9% of the RTX 3060's 490 GB/s peak. A float4 vectorization variant adds another 2.4% improvement (to 92.8% utilization), though the gain is marginal because the coalesced row-major access pattern already achieves full 128-byte cache line transactions. The warp shuffle variant is a special-case optimization for D <= 32 where a single warp handles the entire row.

Current status: all variants pass correctness verification against a host-side reference implementation, with cross-variant differences within 1e-7 (float32 precision). The kernel is confirmed memory-bound with a compute intensity of ~0.2 FLOP/byte (vs the 35.3 FLOP/byte ridge point), meaning all future optimizations must target HBM traffic reduction. Next steps include acquiring ncu hardware counters on bare metal and extending to half-precision formats.
